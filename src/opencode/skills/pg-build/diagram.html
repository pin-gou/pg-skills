判定条件说明（基于实际代码实现）

Pipeline 顺序：由 get_pipeline_order() 从 config.yaml stages[].tracks 生成。当前配置只有 dev stage（tracks: backend → agent → openapi-gen → frontend 顺序执行）。每个 track 内按 test → dev → verify → gate 依次推进。

Phase 项：prepare_env/clean_env 环境生命周期 hook + type: simple track 不走 TDVG，由 _execute_phase() 直接运行命令或派 pg-build/simple agent 执行。

Final Gate：所有 pipeline item 完成后进入，agent 检查跨 track 依赖项（design.md 🆕 标记）→ PASS 则 auto-archive + done，FAIL 则 workflow_failed。
